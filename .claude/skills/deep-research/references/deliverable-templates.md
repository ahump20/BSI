# Deliverable Templates

All deliverables follow one rule: report in plain English. No jargon. Lead with the answer. Support with evidence. Save to a file — research that lives only in chat gets lost.

## Research Brief (Default)

Use for any open-ended research question.

```markdown
# [Research Question — Precise Formulation]

## Executive Summary
[3-5 sentences. The answer upfront. Confidence level stated. Self-contained —
readable without the full brief.]

## Key Findings

### [Sub-question 1]
[Finding with source attribution. Confidence: HIGH / MEDIUM / LOW.]

### [Sub-question 2]
[Finding with source attribution. Confidence: HIGH / MEDIUM / LOW.]

[Continue for each sub-question from Phase 1 decomposition]

## Contradictions & Open Questions
[Where sources disagreed. What remains unresolved. What would resolve it.]

## Evidence Quality Assessment
[What we're confident about vs. what's provisional. Source quality notes.]

## Source Appendix
[Every source used, with quality rating and what it contributed]
```

## Comparative Matrix

Use when the request is "Compare A vs B" or "A vs B vs C."

```markdown
# [Comparison Title]

## Bottom Line
[1-3 sentences. Which option wins and why. State the criteria that drove the call.]

## Comparison Matrix

| Dimension | [Entity A] | [Entity B] | [Entity C] | Edge |
|-----------|-----------|-----------|-----------|------|
| [Criterion 1] | [Evidence] | [Evidence] | [Evidence] | [Winner] |
| [Criterion 2] | [Evidence] | [Evidence] | [Evidence] | [Winner] |

## Analysis by Dimension
[For each row in the matrix — the evidence behind the rating, source attribution,
and why the edge was awarded.]

## Caveats
[What this comparison doesn't capture. Context-dependent factors.]

## Sources
[Every source, organized by entity]
```

## Decision Brief

Use when the request is "Should we do X?" or implies a decision.

```markdown
# [Decision Question]

## Recommendation
[Clear position. One sentence. Then the reasoning.]

## Options Evaluated

### Option 1: [Name]
- **Case for:** [Evidence-backed argument]
- **Case against:** [Evidence-backed counterargument]
- **Risk:** [What could go wrong]

### Option 2: [Name]
[Same structure]

## Key Evidence
[The findings that most influenced the recommendation. Source attribution.]

## What Would Change the Answer
[Conditions under which a different option would be better.
This prevents the recommendation from becoming stale.]

## Sources
```

## Academic Synthesis

Use for literature review requests or research grounded in scholarly sources.

```markdown
# [Topic] — Literature Review

## State of Knowledge
[What the field currently knows, with consensus level indicated.]

## Key Studies

### [Author Year] — [Title]
- Method: [Study design, sample size]
- Finding: [Core result with effect size if available]
- Limitation: [Most important caveat]
- Relevance: [Why this study matters for the research question]

[Repeat for 5-10 most important studies]

## Methodological Notes
[Common study designs in this area. What makes evidence strong vs. weak here.]

## Gaps in the Literature
[What hasn't been studied. What would advance understanding.]

## Sources
```

## Competitive Intelligence Report

Use for market analysis, company research, or competitive positioning questions.

```markdown
# [Subject] — Competitive Intelligence

## Executive Summary
[Market position, key differentiators, trajectory.]

## Company/Product Profiles

### [Entity 1]
- **Position:** [Market standing]
- **Strengths:** [Evidence-backed]
- **Weaknesses:** [Evidence-backed]
- **Recent moves:** [Last 6-12 months]
- **Trajectory:** [Where they're headed]

[Repeat for each entity]

## Market Dynamics
[Forces shaping the competitive landscape. Trends. Shifts.]

## Implications for BSI
[What this means for our positioning, product, or strategy.]

## Sources
```

## Editorial Research Package

Use when research feeds into an article, presentation, or content piece.

```markdown
# Research Package: [Topic]

## Thesis Support
[The core argument the evidence supports. Ready to build an article around.]

## Key Facts & Data Points
[Numbered list. Each with source. Ready to drop into prose.]

## Quotes & Attributions
[Notable quotes from sources. Properly attributed.]

## Counter-Arguments
[The strongest case against the thesis. Why it doesn't hold — or where it does.]

## Narrative Threads
[Story angles the evidence supports. Human interest. Surprising findings.]

## Sources
```
