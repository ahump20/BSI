Our Commitment: Blaze Sports Intel is committed to ensuring digital accessibility for all users, including those with disabilities. We continuously improve the user experience and apply relevant accessibility standards to ensure our sports intelligence platform is usable by everyone.

1. ACCESSIBILITY STANDARDS

Blaze Sports Intel strives to conform to Level AA of the Web Content Accessibility Guidelines (WCAG) 2.1, an internationally recognized standard developed by the World Wide Web Consortium (W3C).

Our Compliance Targets:

WCAG 2.1 Level AA Section 508 ADA Title III Texas Accessibility Standards

2. ACCESSIBILITY FEATURES

2.1 Perceivable Content

  • Text Alternatives: All images, charts, and 3D visualizations include descriptive alt text
  • Captions: Video content includes closed captions (when applicable)
  • Color Contrast: Minimum 4.5:1 contrast ratio for normal text, 3:1 for large text (WCAG AA)
  • Resizable Text: Text scales up to 200% without loss of functionality
  • Adaptable Layout: Content structure preserved when CSS disabled
  • Distinguishable Information: Color is not the sole means of conveying information

2.2 Operable Interface

  • Keyboard Navigation: Full keyboard access without mouse (Tab, Shift+Tab, Enter, Space, Arrow keys)
  • Skip Links: "Skip to main content" link for screen reader users
  • Focus Indicators: Visible focus outlines on interactive elements
  • No Keyboard Traps: Focus can move away from all components
  • Timing Adjustable: Live score updates can be paused or customized
  • Seizure Prevention: No flashing content exceeding 3 flashes per second
  • Accessible Navigation: Consistent navigation across pages, breadcrumbs, landmarks

2.3 Understandable Content

  • Readable Text: Minimum 16px font size, clear typography (Inter font family)
  • Language Declaration: HTML lang attribute set to "en" (English)
  • Predictable Behavior: Navigation and functionality consistent across site
  • Input Assistance: Form labels, error messages, and validation clearly identified
  • Error Prevention: Confirmation dialogs for critical actions (account deletion, data export)
  • Plain Language: Sports terminology explained with tooltips/glossary

2.4 Robust Compatibility

  • Valid HTML: Clean, semantic HTML5 markup
  • ARIA Labels: Proper ARIA roles, states, and properties for dynamic content
  • Screen Reader Support: Tested with JAWS, NVDA, VoiceOver, TalkBack
  • Browser Compatibility: Chrome, Firefox, Safari, Edge (latest 2 versions)
  • Mobile Accessibility: Responsive design, touch target size ≥44x44px

3. ASSISTIVE TECHNOLOGY SUPPORT

3.1 Screen Readers

Blaze Sports Intel is compatible with the following screen readers:

  • JAWS (Windows) - Latest version
  • NVDA (Windows) - Latest version
  • VoiceOver (macOS, iOS) - Latest version
  • TalkBack (Android) - Latest version
  • Narrator (Windows) - Latest version

3.2 Browser Extensions & Tools

  • Screen Magnifiers: ZoomText, MAGic (Windows), Zoom (macOS)
  • Voice Control: Dragon NaturallySpeaking, Voice Control (macOS/iOS)
  • High Contrast Mode: Windows High Contrast themes supported
  • Browser Zoom: Up to 200% zoom without horizontal scrolling

4. MOBILE ACCESSIBILITY

4.1 Mobile-First Design

Our platform is optimized for mobile devices, particularly iPhone portrait mode (375-428px width):

  • Touch Targets: Minimum 44x44px for buttons and links
  • Gestures: Single-tap primary interactions (no multi-finger gestures required)
  • Portrait Orientation: Full functionality in portrait mode (landscape optional)
  • Screen Reader Compatibility: VoiceOver (iOS) and TalkBack (Android) tested
  • Dark Mode: Supports iOS/Android system-level dark mode preferences

5. KNOWN LIMITATIONS

Areas for Improvement:

  • 3D Visualizations: Babylon.js stadium models have limited screen reader support (alternative text descriptions provided)
  • Live Data Tables: Rapidly updating sports scores may be difficult for screen readers (pause button available)
  • AI Copilot: Voice input not yet supported (planned for 2026)
  • Video Analysis: Auto-generated video highlights lack audio descriptions (manual descriptions added within 48 hours)
  • Historical Archives: Some older articles (pre-2024) may not meet current WCAG 2.1 AA standards (being retroactively updated)

6. ACCESSIBILITY TESTING

6.1 Testing Methods

We employ multiple testing approaches to ensure accessibility:

  • Automated Testing: Lighthouse, axe DevTools, WAVE (monthly audits)
  • Manual Testing: Keyboard-only navigation, screen reader testing (quarterly)
  • User Testing: Feedback from users with disabilities (ongoing)
  • Third-Party Audits: Annual WCAG 2.1 AA compliance audit by certified experts

6.2 Continuous Improvement

We regularly test and improve accessibility:

  • Quarterly Reviews: Accessibility audit of new features
  • User Feedback Loop: Dedicated accessibility feedback email
  • Staff Training: Annual accessibility training for all developers and designers
  • Accessibility Champions: Designated team members responsible for compliance
